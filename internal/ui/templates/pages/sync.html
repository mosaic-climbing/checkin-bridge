Sync & Jobs

Manually trigger background data syncs and review recent runs.

Cache Sync
…
Recounts how many members are Active / Frozen / Expired based on what's currently in the local mirror. Read-only — does not talk to Redpoint or UA-Hub. Safe to run any time.
Status Sync
…
Pushes each mapped member's current status (Active / Frozen / Expired) to UA-Hub so frozen members can't scan in. Runs hourly in the background; click here to force an immediate pass. Writes to UA-Hub.
Directory Sync
…
Re-pulls the full Redpoint customer directory (~82k rows) into the local mirror. Runs nightly; trigger here after a big Redpoint edit. Read-only against UA-Hub; takes a few minutes.
UniFi Ingest
…
Reads every UA-Hub user with an NFC tag, matches them to a Redpoint customer by email, and writes the resulting card ↔ customer mapping into the cache. "Dry Run" shows what would change without writing; "Apply" commits.

Unmatched UniFi Users

UniFi accounts with NFC tags that couldn't be auto-paired with a Redpoint customer on the last ingest. Fix the name/email in Redpoint (or match manually in the Needs Match tool), then re-run UniFi Ingest above.
