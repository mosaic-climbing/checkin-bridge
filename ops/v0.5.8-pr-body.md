# v0.5.8: WS reconnect wedge + split-DB guard + /health bypass + Members Select button

Four-ticket ops batch that closes the open threads since v0.5.7.1:

1. **#76 — WebSocket reconnect loop wedged under a dead socket.**
   `readLoop`'s defer waited unconditionally on `pingDone`. The ping
   goroutine took `connMu` and called `conn.WriteControl`; on a dead
   socket that `WriteControl` blocks indefinitely while still holding
   `connMu`, so `readLoop`'s `<-pingDone` never returned, `connectLoop`
   never advanced to its reconnect step, and the bridge's WebSocket
   went dark until the process was restarted. Seen at LEF during the
   0.5.x WS flap — symptom was zero check-ins for ~9h with a
   "connected" health status.

   Fix is three layers so one regression doesn't re-wedge the loop:

   - **Scoped `pingCtx`** (child of `readLoop`'s ctx) so a
     well-behaved ping goroutine observes `pingCtx.Done()` and exits
     immediately on `readLoop` return instead of waiting for the next
     tick.
   - **Bounded wait** in `readLoop`'s defer: if the ping goroutine
     is stuck inside `WriteControl`-under-`connMu` and *can't*
     observe `pingCtx.Done()`, `readLoop` abandons the wait after
     `pingDoneGraceTimeout` (2s) and returns so `connectLoop`'s
     reconnect step can fire.
   - **Panic recovery in `connectLoop`**: any panic in dial,
     `readLoop`, or a user callback re-spawns `connectLoop` instead
     of silently killing the reconnect loop.

   Real bug caught by the regression tests during development: the
   original fix used two separate defers — `defer pingCancel()` and
   `defer` for the bounded wait. LIFO defer ordering meant the
   bounded wait ran *before* `pingCancel`, defeating layer 1 on the
   happy path. Combined into a single defer with explicit ordering
   (cancel first, then wait) once the `PingCtxCancelledOnReturn`
   test flagged the 10s stall.

2. **#80 — `splitLegacyDB` refused to start on fresh installs.**
   The split-legacy guard checked `schema_version` and bailed if it
   was at or past the target, but on a brand-new install
   `schema_version` is pinned at target from migration — with no
   legacy table to split. Result: fresh installs failed boot with a
   misleading "already split" error. Fix: before comparing versions,
   verify the legacy table actually exists via `sqlite_master`. No
   legacy rows → no split to do → skip the guard entirely.

3. **#93 — `/health` 403'd behind `AllowedNetworks`.**
   The deploy script and external uptime checks hit `/health` from
   off-LAN addresses and got 403 from the IP allowlist middleware.
   `/health` is intentionally un-authenticated and returns nothing
   sensitive, so the right answer is to carve it out ahead of the
   allowlist rather than teach every caller to hold a token. Fix:
   path-scoped carve-out in the middleware before the IP check.
   Added regression test so the carve-out doesn't quietly regress
   and expose other endpoints.

4. **#118 — Members page "Select" button silently did nothing for
   any name or Redpoint ID containing `'` or `"`.**
   `SearchResultsFragment` rendered an inline `onclick` with the
   name and Redpoint ID embedded as JS string literals between
   single quotes. `HTMLEscape` turned apostrophes/double-quotes into
   numeric entity references (`&#39;`, `&#34;`), the HTML attribute
   parser decoded those back to literal `'`/`"` inside the `onclick`
   value, and the embedded JS parser then saw a string literal
   terminated mid-name — e.g. `value='O'Brien'` — and raised a
   `SyntaxError` that left the button silently doing nothing. Names
   like O'Brien, O'Connor, D'Angelo, etc. all broke. Fix moves the
   payload onto `data-*` attributes and puts the click logic in a
   delegated handler in `members.html`. Data attributes are safe
   under `HTMLEscape` because `getAttribute` returns the raw decoded
   string — the JS never has to quote it.

   Also fixed a secondary bug while there: the old handler dropped
   the full name into the `firstName` field. Now splits on the first
   space, so "Mary Jane Smith" → first="Mary", last="Jane Smith",
   matching what a human would type into the form.

## What ships

### #76 — WS reconnect wedge (three-layer fix)

- `internal/unifi/client.go`:
  - New package-level `wsPingInterval` var (was inline const) and
    `pingDoneGraceTimeout` — both overridable from tests.
  - `readLoop`: `pingCtx, pingCancel := context.WithCancel(ctx)`
    scopes the ping goroutine to the read loop's lifetime. Ping
    goroutine's select now watches `pingCtx.Done()` alongside the
    ticker.
  - `readLoop`'s defer is a single combined block: `pingCancel()`
    first, then `select { case <-pingDone: case <-time.After(
    pingDoneGraceTimeout): logger.Warn(...) }`. Explicit ordering
    prevents the LIFO-ordering footgun.
  - `connectLoop`: wraps each iteration in a defer/recover that
    logs the panic and re-spawns `connectLoop(ctx)` on a fresh
    goroutine. Parent ctx is honored — a cancelled ctx won't
    respawn.

### #80 — splitLegacyDB pin-on-fresh-DB

- `internal/store/store.go`: `splitLegacyDB` now runs a
  `SELECT name FROM sqlite_master WHERE type='table' AND name=?`
  probe against the legacy table before comparing schema versions.
  Fresh install → probe returns no row → short-circuit, no error.
  Comment block explains the v0.5.0 deploy failure that drove the
  change.

### #93 — /health carve-out

- `internal/api/middleware.go`: IP-allowlist middleware now
  short-circuits for `r.URL.Path == "/health"` before consulting
  `AllowedNetworks`. Scoped to exact-path match so a future
  `/health/foo` doesn't accidentally inherit the carve-out.
  Comment captures the deploy-script and uptime-check use cases.

### #118 — Members Select button

- `internal/ui/fragments.go`: `SearchResultsFragment` emits the
  Select button as
  `<button type="button" class="btn btn-success btn-sm"
  data-action="select-member" data-redpoint-id="..."
  data-name="...">Select</button>`.
  Both payload fields go through `HTMLEscape`; no inline `onclick`.
- `internal/ui/templates/pages/members.html`: adds a delegated
  `click` listener that reads `data-redpoint-id`/`data-name`,
  splits the name on the first space, fills the Add-Member form,
  and scrolls+focuses the NFC UID input. Comment block pins why
  data attributes avoid the HTMLEscape/JS-literal class of bug.

## Tests

### `internal/unifi/reconnect_wedge_test.go` (new)

- `TestReadLoop_BoundedWaitOnWedgedPingGoroutine` — primary #76
  regression. Holds `c.connMu` from the test to force the ping
  goroutine to block on `Lock()` (standing in for the prod
  `WriteControl`-under-`connMu` wedge), triggers a read error, and
  asserts `readLoop` returns within `pingDoneGraceTimeout` + slack.
  Pre-fix: blocks forever.
- `TestReadLoop_PingCtxCancelledOnReturn` — happy-path half. Uses
  `wsPingInterval=10s` so the ticker can't save us; proves the
  scoped `pingCtx.Done()` path exits the ping goroutine promptly.
  This is the test that caught the LIFO-defer bug during dev.
- `TestConnectLoop_PanicRecoveryRespawns` — panics from inside a
  user `onStateChange(Connecting)` callback via `sync.Once`, then
  cancels ctx and asserts the respawned goroutine fires
  `setState(Disconnected)` on ctx.Done. Pre-fix: the original
  goroutine dies, no respawn, no Disconnected callback.

### `internal/api/health_bypass_test.go` (new)

- Full matrix: `/health` from an allow-listed IP, `/health` from a
  blocked IP, and `/health-ish` paths (`/healthx`, `/health/sub`)
  from a blocked IP to confirm the carve-out is exact-match and
  doesn't leak. Plus `/metrics` from a blocked IP to pin that
  other endpoints still 403.

### `internal/store/split_malformed_test.go` (new)

- Creates a fresh database at target `schema_version`, with no
  legacy `checkins_legacy` table, and asserts `Open` succeeds and
  `RecordCheckIn` works end-to-end. Pre-fix: `Open` returned an
  error. Uses `&CheckInEvent{...}` (struct is `CheckInEvent`,
  passed by pointer).

### `internal/ui/fragments_test.go` (new)

- `TestSearchResultsFragment_SelectButtonUsesDataAttrs` — positive
  contract: `data-action="select-member"`, `data-redpoint-id`,
  `data-name` all present. Negative contract: no `onclick=` in the
  rendered output.
- `TestSearchResultsFragment_NameWithApostropheRendersSafely` —
  the `"Sean O'Brien"` case that broke pre-fix. Asserts
  `data-name="Sean O&#39;Brien"` and no `onclick=`.
- `TestSearchResultsFragment_QuoteInIdEscapedSafely` — defensive
  pin for a `"` in the Redpoint ID. Asserts
  `data-redpoint-id="cust_&#34;exotic&#34;"`.
- `TestSearchResultsFragment_EnrolledRowHasNoSelectButton` —
  unchanged contract: once a row is enrolled, no Select button.
  Plus `Enrolled (04AABB1122)` badge still carries the NFC UID.
- `TestSearchResultsFragment_EmptyResults` — friendly
  `"No results found"` placeholder on empty input.
